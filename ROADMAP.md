# Waykin MPOC Roadmap

## Phase 0 — Baseline

- Generate the iOS project
- Select minimum iOS and Xcode versions
- Confirm simulator build
- Add a first passing test target

Exit: reproducible clean build and test command.

## Phase 1 — Domain and Movement Kernel

- Typed domain models
- Clock, location, and motion protocols
- Session start/pause/resume/end
- Distance, pace, and route calculations
- Deterministic simulated movement

Exit: movement tests pass under fixed input.

## Phase 2 — Experience Runtime

- Experience registry and contract
- Presentation-neutral updates
- Companion command model
- Companion Walk implementation

Exit: Calm Day Walk completes in Demo Mode.

## Phase 3 — Distinct Mechanics

- Orc Pursuit state and bounded pressure rules
- Future Self target and catch-window rules
- Day/night variants
- Attention and pause behavior

Exit: all three deterministic scenarios complete.

## Phase 4 — Persistence and Continuity

- Local companion/session storage
- Bond progression
- Deterministic memory generator
- Prior-pace retrieval
- Relaunch continuity tests

Exit: continuity proof passes.

## Phase 5 — Recommendation and Presentation

- Controllable time context
- Explained recommendation scoring
- Map presentation
- Audio cues
- Permission-denial behavior

Exit: day/night recommendations differ and no permission is required for Demo Mode.

## Phase 6 — AR Spike

Attempt a minimal RealityKit companion only after Phases 0–5 are stable. Revert or isolate the spike if it harms reliability.

## Phase 7 — Validation and Receipt

- Full build and tests
- Five-minute demo
- Documentation reconciliation
- Known limitations
- Final evidence-based status

No phase may claim completion without observed validation.
